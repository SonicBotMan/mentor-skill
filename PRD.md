# 导师.SKILL — PRD v0.2

> **"导师可以走，但她的判断力可以留下。"**

---

## 1. 背景与问题

### 1.1 职场导师的稀缺性

职场中最有价值的学习资源不是文档，是**导师**。导师的价值在于：

- 她追问问题的方式，让你学会自己找答案
- 她的判断框架，让你少走弯路
- 她的知识边界，告诉你什么该深入、什么该放弃
- 她记得你的项目，会主动追问进展

但导师资源是稀缺的：转岗、离职、升职、忙碌，随时可能中断指导关系。

### 1.2 现有 AI 助手的局限

现有 AI 助手（ChatGPT、Claude 等）在"导师模拟"场景的核心问题：

| 问题 | 表现 |
|------|------|
| 无个性化 | 每次对话都是"空白状态"，不记得你的项目 |
| 无指导风格 | 倾向于直接给答案，而非引导思考 |
| 无领域深度 | 什么都会一点，没有专家级深度 |
| 无关系连续性 | 无法追问"上次我说的那个建议你想通了吗？" |

### 1.3 机会

**导师.SKILL** 通过把真实导师的沟通记录蒸馏成结构化的 AI Skill，将她的思维方式、反馈风格、知识体系、甚至口头禅全部保留，并持久化为跨平台可用的 AI 角色卡。

---

## 2. 目标用户

### 主要用户：职场学徒

- 正在接受导师指导，想把指导内容"存档"
- 导师即将离开，想保留她的判断力
- 没有接触到优秀导师的机会，想通过蒸馏他人经验学习

### 次要用户：导师本人

- 愿意把自己的方法论固化成 AI Skill，扩大影响力
- 希望减少重复性的指导工作（基础问题让 AI 回答）

## 3. 核心功能

### 3.1 7 层 Persona 模型

```
Layer 7: 学徒记忆层  — 记得你的项目、追问进展、检查修改  ⭐ 核心差异
Layer 6: 指导关系层  — 她怎么带人、怎么给反馈
Layer 5: 情感表达层  — 她的情绪和共情方式
Layer 4: 沟通风格层  — 她怎么说（语气、节奏、修辞）
Layer 3: 思维框架层  — 她怎么想（方法论、决策逻辑）
Layer 2: 知识与专业层— 她知道什么（领域、边界、深度）
Layer 1: 基础身份层  — 她是谁（角色、背景、性格）
```

**Layer 7（学徒记忆层）** 是与同类项目的核心差异：AI 导师会记得你的具体项目，主动追问进展，检查你是否采纳了建议。

### 3.2 多数据源采集

| 平台 | 采集方式 | 状态 |
|------|---------|------|
| Markdown/文本 | 本地文件解析 | ✅ |
| PDF | pdfplumber 解析 | ✅ |
| 微信 | SQLite/CSV 导出 | ✅ |
| 飞书 | 开放平台 API | ✅ |
| 钉钉 | API + Playwright 浏览器 | ✅ |
| 如流 | API（规划中） | 🔴 |

### 3.3 蒸馏引擎

- 使用 LLM 对原始对话数据逐层分析，提取 Persona 特征
- 支持交互式确认（每层结果可手动修改或重新蒸馏）
- 断点恢复：每层完成后保存 checkpoint，中断后可续跑

### 3.4 多平台 Skill 输出

| 平台 | 格式 | 文件 |
|------|------|------|
| 通用 | Markdown | `.SKILL.md` |
| Cursor IDE | MDC Rules | `.mdc` |
| Claude Code | SKILL.md | `.claude.SKILL.md` |
| OpenClaw | AgentSkills | `.openclaw.skill.md` |

### 3.5 对话与验收

- `mentor test`：与蒸馏后的 Persona 多轮对话，验证效果
- `mentor demo`：内置 Sample Persona，无需数据立即体验
- `evals/run_eval.py`：LLM-as-judge 自动评估 Skill 质量（三维评分）
